**Section 9: Temporal intensity computation (H3).** For each of the 34 points, compute the temporal intensity profile of (a) the unshaped pulse and (b) the shaped pulse:

```julia
for point in PA_all_points
    Nt = Int(point["Nt"])
    uomega0 = vec(point["uomega0"])  # FFT-order complex field
    phi_opt = vec(point["phi_opt"])    # FFT-order phase

    # Unshaped: just IFFT the input field
    ut_unshaped = ifft(uomega0)
    I_unshaped = abs2.(ut_unshaped)

    # Shaped: apply phi_opt then IFFT
    ut_shaped = ifft(uomega0 .* cis.(phi_opt))
    I_shaped = abs2.(ut_shaped)

    # Time axis in ps (sim_Dt is in ps)
    sim_Dt_ps = Float64(point["sim_Dt"])
    t_ps = fftshift((0:Nt-1) .* sim_Dt_ps .- (Nt * sim_Dt_ps / 2))

    # fftshift the intensities for plotting
    I_unshaped_shifted = fftshift(I_unshaped)
    I_shaped_shifted = fftshift(I_shaped)

    # Store metrics
    point["I_unshaped"] = I_unshaped_shifted
    point["I_shaped"] = I_shaped_shifted
    point["t_ps"] = t_ps
    point["peak_power_ratio"] = maximum(I_shaped) / maximum(I_unshaped)
    point["temporal_rms_ratio"] = sqrt(sum(I_shaped.^2)) / sqrt(sum(I_unshaped.^2))
end
```

Key metrics per point:
- `peak_power_ratio` = max(I_shaped) / max(I_unshaped) — how much does the optimizer reduce peak power?
- `temporal_spread_ratio` = rms_width(I_shaped) / rms_width(I_unshaped) — how much does it stretch the pulse?
- `effective_N_shaped` = N_sol * sqrt(peak_power_ratio) — the effective soliton number after shaping (approximate, since N is technically invariant under phase-only shaping, but the effective peak power at the fiber input changes the initial dynamics)

**Section 10: Raman response function and overlap integral (H7).** Compute the Raman response function h_R(t) for silica using the standard Blow & Wood (1989) damped oscillator model, then compute the overlap integral with the shaped/unshaped temporal intensity.

The Raman response in time domain:
```julia
# Raman parameters (silica, from helpers.jl line 100)
const PA_FR = 0.18       # fractional Raman contribution
const PA_TAU1 = 12.2     # fs (oscillation period)
const PA_TAU2 = 32.0     # fs (decay time)

# For each point, build h_R(t) on that point's time grid:
function compute_raman_response(t_ps, Nt)
    t_fs = t_ps .* 1e3  # ps -> fs
    # Causal: only t >= 0
    t_pos = max.(t_fs, 0.0)
    h_R = (PA_TAU1^2 + PA_TAU2^2) / (PA_TAU1 * PA_TAU2^2) .*
           exp.(-t_pos ./ PA_TAU2) .* sin.(t_pos ./ PA_TAU1) .*
           (sign.(t_fs) .+ 1) ./ 2
    # Normalize so integral = 1
    dt_fs = abs(t_fs[2] - t_fs[1])
    h_R ./= (sum(h_R) * dt_fs + 1e-30)
    return h_R
end
```

The Raman gain is proportional to the convolution of |E(t)|^2 with h_R(t). The overlap integral that quantifies how strongly the shaped pulse drives the Raman process:

```julia
# Raman overlap integral (frequency domain is faster for convolution):
# G_R = integral{ |H_R(Omega)| * S_I(Omega) dOmega }
# where S_I = |FFT(|E(t)|^2)|^2 is the PSD of the temporal intensity,
# and H_R = FFT(h_R) is the Raman response in frequency domain.
#
# Physical meaning: G_R measures how much spectral weight of the
# temporal intensity fluctuations overlaps with the Raman gain bandwidth.

function raman_overlap_integral(I_t, h_R, dt_ps)
    # PSD of intensity in frequency domain
    S_I = abs2.(fft(I_t))
    H_R = abs.(fft(h_R))

    # Frequency grid
    df = fftfreq(length(I_t), 1.0 / dt_ps)  # THz

    # Overlap integral (spectral, unnormalized)
    G_R = sum(S_I .* H_R) * (df[2] - df[1])
    return G_R
end
```

Compute for each point:
- `G_R_unshaped` = raman_overlap(I_unshaped, h_R) — baseline Raman drive
- `G_R_shaped` = raman_overlap(I_shaped, h_R) — shaped Raman drive
- `G_R_ratio_dB` = 10*log10(G_R_shaped / G_R_unshaped) — reduction in dB

The coherent interference hypothesis predicts: G_R_ratio_dB should correlate strongly with delta_J_dB. If the optimizer's main mechanism is reducing the Raman overlap, these should track each other.

## Risks

1. **Temporal intensity from IFFT may need fftshift.** The time-domain pulse from `ifft(uomega0)` is in FFT order (t=0 at index 1). Must `fftshift` for plotting. The intensity computation `abs2` is order-independent, but the convolution with h_R must account for causal ordering. Mitigation: build h_R on the same FFT-order time grid, compute overlap in frequency domain (no ordering issues).

2. **Raman overlap integral normalization.** G_R values may span many orders of magnitude across configs (different Nt, time windows). Mitigation: always report as a RATIO (shaped/unshaped), so normalization cancels.

3. **Group delay noise at signal band edges.** Central differencing amplifies noise where phi_opt transitions to the noise floor. Mitigation: apply the -40 dB signal mask and set masked bins to NaN (same as physics_insight.jl Figure 6).

4. **H5 (propagation diagnostics) deferred.** The research recommended testing where Raman energy appears along z. This requires re-propagation with intermediate z snapshots, which contradicts the "no re-running" constraint. H5 is out of scope for this plan. If the temporal/overlap analysis is conclusive, H5 is unnecessary. If inconclusive, a follow-up plan could add re-propagation.
