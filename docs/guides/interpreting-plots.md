# Interpreting Plots

[← docs index](../README.md) · [project README](../../README.md)

This guide is for reading the standard figures produced by canonical runs and
for recognizing the common failure modes that show up in sweep outputs.

## Color conventions (Okabe-Ito)

- **Blue** = input (before fiber).
- **Vermillion / orange** = output (after fiber).
- **Green** = optimized output (when distinguished from flat-phase output).
- **Purple / magenta** = reference / theoretical.
- **Gray** = noise floor / annotation.

Inherited from the v1.0 visualization overhaul — the Okabe-Ito palette is
colorblind-safe and the conventions are consistent across every plot in the
repo. See `.planning/milestones/v1.0-ROADMAP.md` for the palette decision.

## Spectral plots

X-axis: wavelength in nm (or optionally frequency in THz). Y-axis: power
spectral density in dB (normalized to input peak = 0 dB).

Features to notice:
- Shaded region: Raman gain band (~13 THz below the pump).
- Input (blue) should be a clean Gaussian centered at λ₀.
- Output (vermillion) shows broadened spectrum with Raman feature visible or
  suppressed depending on optimization state.
- **Good suppression**: output curve drops 30+ dB inside the shaded band
  relative to the flat-phase baseline. 50+ dB is excellent.

## Phase plots (3-view)

Three panels share an X-axis (frequency, THz relative to carrier):

1. **Wrapped phase** (mod 2π): shows the actual phase mask the optimizer
   produced. Expect fringes and discontinuities.
2. **Unwrapped phase**: smooth version after masking out noise-dominated
   frequencies.
3. **Group delay** (−dφ/dω in fs): the physical quantity you'd program into a
   dispersion compensator.

Mask-before-unwrap: only frequencies inside the signal-bearing region are
unwrapped. Everything outside is zeroed.

## Evolution heatmaps

2-panel (spectral evolution, temporal evolution) × 2 columns (before / after).
X = z-coordinate along fiber (m); Y = frequency (THz) or time (ps); color =
power in dB.

- `inferno` colormap (dark = low power, bright = high).
- **Good suppression**: post-optimization spectral evolution shows NO bright
  feature growing inside the Raman band as z increases.

## Convergence plot

Y-axis: J in dB (converted post-optimization via `lin_to_dB`).
X-axis: L-BFGS iteration.

- Monotonically decreasing is expected.
- Plateau after iteration 20–30 is normal — sweeps use `max_iter=30`.
- A flat line at -3 to -5 dB with no descent means the optimizer is failing;
  see troubleshooting in
  [quickstart-optimization.md](./quickstart-optimization.md#troubleshooting).

## Heatmaps (sweep)

L × P grid colored by `J_final_dB`.

- Smooth, monotonic variation is the healthy pattern: longer L and higher P
  give deeper suppression until a saturation boundary.
- Bright patches in an otherwise smooth field usually indicate (a) a non-
  converged point (check the sidecar `converged` field — see
  [output-format.md](../architecture/output-format.md)), or (b) a photon-number-drift
  point (check the JLD2 metadata for drift > 5%).

## Common artifacts

| Artifact | Likely cause |
|----------|--------------|
| Spectral ringing at window edges | Time window undersized relative to dispersive walk-off + SPM broadening. Increase `time_window_ps`. |
| Noise-like structure in unwrapped phase | Mask threshold too permissive. The plotter masks below a power threshold before unwrapping; adjust in `scripts/lib/visualization.jl`. |
| Convergence history shows NaN late | Numerical instability — reduce `Nt`, tighten tolerances, or lower `P_W`. |
| Evolution colormap all dark | Power normalization is off; check the P_ref normalization call-site in `scripts/lib/visualization.jl`. |
