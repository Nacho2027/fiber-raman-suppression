# Public Release Readiness

The repo is not ready for a broad public release just because the local smoke
passes.

## Before public release

- trim private infrastructure assumptions from docs;
- publish only intentional fixtures and figures;
- make external data provenance clear;
- verify install instructions on a clean machine;
- document exactly which workflows are supported;
- choose a small example result that can be regenerated;
- confirm license and attribution text.

## Reasonable first release

A narrow preview release around the single-mode phase command is realistic.
MMF, long-fiber, and multivariable research should remain marked experimental
unless their gates are closed separately.
