<objective>
Fix the rcParams mutation bug and replace the jet colormap with inferno on all four evolution/spectrogram functions. Also disable grid on pcolormesh heatmap axes (grid lines appear as data artifacts).

Purpose: These are the two most visually damaging rendering bugs. Jet creates ~3 dB false perceptual features (Crameri et al. 2021); the rcParams silent-failure bug means current font/dpi settings may not persist in batch runs. Together these prevent plots from being usable in lab meetings.

Output: `scripts/visualization.jl` with corrected rcParams block, inferno colormap defaults in all four heatmap functions, and ax.grid(false) after every pcolormesh call.
</objective>

```julia
const _rc = PyPlot.PyDict(PyPlot.matplotlib."rcParams")
_rc["font.size"]          = 11
_rc["axes.labelsize"]     = 12
_rc["axes.titlesize"]     = 13
_rc["xtick.labelsize"]    = 10
_rc["ytick.labelsize"]    = 10
_rc["legend.fontsize"]    = 10
_rc["figure.dpi"]         = 150
_rc["savefig.dpi"]        = 300
_rc["savefig.bbox"]       = "tight"
_rc["axes.grid"]          = true
_rc["grid.alpha"]         = 0.3
```

Key points:
- The constant is named `_rc` (underscore prefix = private helper, per project conventions)
- `PyPlot.PyDict(PyPlot.matplotlib."rcParams")` uses the documented-correct Julia syntax with the dot-quoted field access — do NOT use `PyPlot.matplotlib.rcParams` (no quotes), which returns a copy
- `savefig.bbox = "tight"` is a new addition (was missing before — caused label clipping)
- All other values remain unchanged from the original block (font.size=11, axes.labelsize=12, etc.)
- Do NOT add any settings not listed above — no `font.family`, no `lines.linewidth`, no Okabe-Ito prop_cycle (these are Phase 2+ concerns)
