<decisions>
## Implementation Decisions

### Philosophy
- **D-01:** This is DISCOVERY, not storytelling. Design figures that show the raw physics from multiple views so unexpected patterns can emerge. Don't optimize for confirming a narrative — optimize for revealing structure the user hasn't seen yet.

### Phase Profile Visualization
- **D-02:** Two-panel phase profile plots: (a) φ_opt vs frequency offset from ω₀ (THz) with Raman band shaded at ~13 THz, (b) φ_opt vs wavelength (nm) for experimental context. Remove global phase offset and linear group delay term before plotting so only chirp structure remains. Overlay all 5 configs on shared axes to reveal similarities/differences.

### Correlation Analysis
- **D-03:** Claude's discretion on which correlations to plot. Recommended: multi-panel with N, L, P vs ΔdB (3 scatter plots). If N dominates, the other axes will show weak trends — that itself is informative. Color/marker by fiber type.

### Figure Set
- **D-04:** Comprehensive explorer set (~6-8 figures). Cast a wide net:
  1. Raw φ_opt overlay (all 5, freq domain, Raman-annotated)
  2. Raw φ_opt overlay (all 5, wavelength domain)
  3. Per-run φ_opt detail panels (5-panel grid, each with polynomial fit overlaid)
  4. Correlation scatter panel (N/L/P vs ΔdB)
  5. Before/after Raman band comparison per run (input vs optimized output spectra, Raman region zoomed)
  6. Group delay comparison (dφ/dω for each run — shows time-domain reshaping)
  7. Polynomial fit residual visualization (actual phase minus fit, all 5 overlaid)
  8. Phase structure at Raman offset — zoom into φ_opt around the 13 THz Raman band specifically

### Claude's Discretion
- Exact figure sizes and layouts
- Whether to add additional exploratory views if something interesting appears during implementation
- Color palette and labeling
- Whether some figures get combined into multi-panel layouts vs separate files
- Statistical annotations (R², trend lines) on correlation plots

### Phase Normalization Recipe
From pitfalls research: before overlaying φ_opt across runs:
1. Remove global offset: φ_norm = φ_opt - φ_opt[center_index]
2. Remove linear trend (group delay): fit linear term to signal-bearing region, subtract
3. What remains is the physically meaningful chirp structure

<specifics>
## Specific Ideas

- The 99% residual from Phase 6 is the most interesting finding so far. Visualizing what that residual LOOKS like (φ_actual - φ_polynomial_fit) may reveal periodic structure, features at Raman offset, or other physically meaningful patterns.
- Group delay (dφ/dω) shows how the optimizer reshapes the pulse in time. This is often more physically intuitive than the phase itself.
- Zooming into the phase near the Raman band offset (~13 THz from center) may reveal whether the optimizer specifically targets that frequency region.
- With only 5 data points, correlation plots should use markers not trend lines — the eye can see patterns without statistical fitting.

</specifics>
