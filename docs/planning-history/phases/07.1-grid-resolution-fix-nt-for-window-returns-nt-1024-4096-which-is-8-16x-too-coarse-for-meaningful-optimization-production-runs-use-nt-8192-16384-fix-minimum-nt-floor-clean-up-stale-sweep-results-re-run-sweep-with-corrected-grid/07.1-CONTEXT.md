# Phase 07.1 Context: Grid Resolution Fix

## Problem Statement

The Phase 7 parameter sweep used `nt_for_window()` directly, which returns Nt=1024-4096 based solely on temporal resolution requirements (dt_min=10.5 fs). This is 8-16x too coarse for meaningful optimization. Production runs in Phases 5-6 use Nt=8192-16384. A single sweep point at Nt=1024 with max_iter=100 took 4 hours (L=2m, P=0.3W), and the full 36-point grid never completed.

Two compounding issues:
1. **Nt too low**: `nt_for_window()` returns the minimum Nt for temporal resolution, not for optimization quality. The optimizer needs spectral resolution (more frequency bins in the Raman band) to find meaningful phase structure.
2. **max_iter too high**: Phase 6 convergence plots show all runs plateau by iteration 30-40. Running 100 iterations wastes 60-70% of compute time with no improvement.

## Locked Decisions

### D-01: Nt floor of 2^13 (8192) in run_sweep.jl

The `nt_for_window()` function in `common.jl` is correct for its purpose (minimum temporal resolution). It stays as-is. Instead, `run_sweep.jl` applies a floor:

```julia
Nt = max(nt_for_window(time_window), 2^13)
```

Rationale: Production runs use Nt=2^14=16384. Nt=2^13=8192 is a 2x compromise that maintains good spectral resolution while keeping sweep runtime tractable. At Nt=8192, each point should take ~5-15 min instead of ~4 hours.

### D-02: Reduce max_iter from 100 to 30

Phase 6 convergence overlay shows all 5 production runs plateau by iteration 30-40. For a sweep where we need relative J_final values (not absolute minimum), 30 iterations captures >95% of the achievable suppression.

Applied in both `run_fiber_sweep()` and `run_multistart()`.

### D-03: Drop L=10m from SMF-28 grid

The SMF-28 grid becomes [0.5, 1.0, 2.0, 5.0] (4 lengths instead of 5). Reasons:
- At L=10m, the time window becomes very large, forcing Nt=16384+ even at Nt=8192 floor
- L=5m already covers the long-fiber regime where dispersive walk-off dominates
- Each L=10m point would take 30+ min, adding 2+ hours to the sweep for marginal physics insight

Grid becomes: 4x4=16 SMF-28 + 4x4=16 HNLF = 32 points total.

### D-04: Clean stale sweep results

The aborted sweep left 15 SMF-28 JLD2 files in `results/raman/sweeps/smf28/` at Nt=1024-4096. These must be deleted before re-running to avoid mixing low-Nt and high-Nt results.

### D-05: Sweep re-run deferred to user

The corrected sweep (32 points + 10 multi-start) is estimated at 3-8 hours depending on hardware. The code changes are made and verified, but the actual sweep execution is left for the user to kick off manually.
