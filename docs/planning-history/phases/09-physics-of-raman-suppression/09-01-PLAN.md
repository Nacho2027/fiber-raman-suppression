This directly addresses the central question (D-02): universal vs arbitrary.

**Section 2: Extended polynomial decomposition (H1).** For each of the 34 points, fit phi_opt onto polynomial basis orders 2 through 6 using weighted least squares in the signal band. The existing `decompose_phase_polynomial()` only fits order 2+3 (GDD+TOD). Extend the analysis by building a Vandermonde matrix up to order N:

```julia
# For each point:
sim_Dt_s = sim_Dt_ps * 1e-12
omega_shifted = 2pi .* fftshift(fftfreq(Nt, 1.0 / sim_Dt_s))  # rad/s
phi_shifted = fftshift(vec(phi_opt))

# Build signal mask at -40 dB
spec_power = abs2.(fftshift(vec(uomega0)))
P_peak_spec = maximum(spec_power)
dB_mask = 10.0 .* log10.(spec_power ./ P_peak_spec .+ 1e-30)
sig_mask = dB_mask .> -40.0

omega_sig = omega_shifted[sig_mask]
phi_sig = phi_shifted[sig_mask]

# Remove offset + linear term first
A_lin = hcat(ones(length(omega_sig)), omega_sig)
coeffs_lin = A_lin \ phi_sig
phi_detrended = phi_sig .- A_lin * coeffs_lin

# Fit orders 2 through max_order
for max_ord in 2:6
    A_poly = hcat([omega_sig.^k ./ factorial(k) for k in 2:max_ord]...)
    coeffs = A_poly \ phi_detrended
    phi_fit = A_poly * coeffs
    residual_frac = norm(phi_detrended .- phi_fit) / (norm(phi_detrended) + 1e-30)
    explained_var = 1.0 - residual_frac^2  # fraction of variance explained
end
```

Store the full coefficient vectors and explained variance at each order in the point dict. Convert coefficients to physical units: order k coefficient has units s^k, convert to fs^k by multiplying by 1e(15k).

**Section 3: Residual PSD analysis (H2).** After order-6 polynomial subtraction, compute the power spectral density of the residual phase in the signal band. The key test: does the residual PSD show a peak or enhanced power near 13 THz (the silica Raman detuning)? Use zero-padded FFT of the residual for better frequency resolution. The frequency axis for this PSD is a "modulation frequency" axis (how rapidly phi varies with optical frequency), not the optical frequency itself. Compute:

```julia
residual_order6 = phi_detrended .- A_poly_6 * coeffs_6
# Zero-pad to 4x length for spectral resolution
N_pad = nextpow(2, 4 * length(residual_order6))
padded = zeros(N_pad)
padded[1:length(residual_order6)] = residual_order6
# PSD in "modulation frequency" space
# The independent variable is omega (rad/s), so the conjugate variable is time (s)
# But we want to see if there's periodicity at delta_omega = 2pi * 13e12 rad/s
d_omega = omega_sig[2] - omega_sig[1]  # rad/s spacing
psd = abs2.(fft(padded))
mod_freq = fftfreq(N_pad, 1.0 / d_omega)  # conjugate variable: seconds
# Convert to THz modulation: features at period 1/(2pi*13e12) in omega-space
# correspond to peaks at tau = 1/(2pi*13e12) ~ 12.2 fs in the PSD
```

Actually, the physical interpretation: if phi_opt has oscillations with period Delta_omega in angular frequency space, this creates temporal copies/replicas separated by Delta_t = 2*pi / Delta_omega. A 13 THz feature means Delta_omega = 2*pi*13e12 rad/s, corresponding to peaks in the FFT at delay tau = 1/Delta_f = 1/(13e12) ~ 77 fs. Report whether the PSD has structure vs white noise.

## Risks

1. **Heterogeneous Nt across sweep points.** Different (L,P) configs have different Nt (from recommended_time_window), so phi_opt vectors have different lengths and frequency grids. Mitigation: interpolate onto a common grid for cross-correlation. Use the coarsest grid's frequency range as the common domain.

2. **Polynomial fitting instability at high orders.** Order-6 Vandermonde matrices can be ill-conditioned when omega spans many orders of magnitude (rad/s). Mitigation: normalize omega to [-1, 1] before fitting, then convert coefficients back. Use `qr()` decomposition instead of `\` for stability.

3. **Residual PSD interpretation.** The "modulation frequency" axis of the residual PSD is not straightforward. Mitigation: clearly label the conjugate variable and explain the physical interpretation (temporal delay between phase-created pulse replicas).

4. **Multi-start P=0.20W mismatch.** The multi-start was run at P=0.20W (not P=0.30W as originally planned in run_sweep.jl comments). This is correct — the code actually uses P_cont=0.20. Verify by reading the multistart JLD2 P_cont_W field.
