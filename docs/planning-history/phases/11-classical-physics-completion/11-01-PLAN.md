**Section A: Multi-Start Z-Propagation (per D-01, D-02)**

Function `pc_load_multistart_and_propagate(start_idx; n_zsave=PC_N_ZSAVE)`:
- Load from `joinpath(PC_MULTISTART_DIR, "start_$(lpad(string(start_idx), 2, "0"))", "opt_result.jld2")`
- Extract: `phi_opt = vec(data["phi_opt"])`, `L = Float64(data["L_m"])`, `P_cont = Float64(data["P_cont_W"])`, `Nt = Int(data["Nt"])`, `time_window = Float64(data["time_window_ps"])`
- All 10 starts use L=2m, P=0.2W, Nt=8192, time_window=40ps, preset=:SMF28
- Call `setup_raman_problem(L_fiber=L, P_cont=P_cont, Nt=Nt, time_window=time_window, beta_order=3, fiber_preset=:SMF28)` — note `beta_order=3` is REQUIRED per Phase 10 finding
- `deepcopy(fiber)` before setting `fiber["zsave"] = collect(LinRange(0, L, n_zsave))`
- Propagate shaped: `MultiModeNoise.solve_disp_mmf(uomega0 .* exp.(1im .* phi_opt), fiber_shaped, sim)`
- Propagate flat: `MultiModeNoise.solve_disp_mmf(uomega0, fiber_flat, sim)` (per D-02)
- Compute J(z): `Float64[spectral_band_cost(sol["uomega_z"][i, :, :], band_mask)[1] for i in 1:n_zsave]`
- Save immediately per propagation pair to `PC_RESULTS_DIR/multistart_start_XX_{shaped,unshaped}_zsolved.jld2` with keys: `uomega_z`, `J_z`, `zsave`, `phi_opt`, `Nt`, `L_m`, `P_cont_W`, `fiber_name`, `sim_Dt`
- Return named tuple: `(J_z_shaped, J_z_flat, zsave, phi_opt, start_idx)`

Main loop: iterate `start_idx in 1:10`, collect all `J_z_shaped` and `J_z_flat` vectors.

**Consistency check (per D-02):** After all 10 flat propagations, assert `maximum(abs.(J_z_flat[i] - J_z_flat[1])) < 1e-10` for all i=2:10. All flat propagations must be identical because they use the same fiber, pulse, and flat phase. Log the max deviation.

- Save to `PC_RESULTS_DIR/multistart_trajectory_analysis.jld2` with keys: `jz_corr_matrix`, `phi_corr_matrix`, `all_jz_shaped_dB`, `zsave`, `J_final_dB` (vector of 10 final J values)

**Section C: Spectral Divergence (per D-04, D-05)**

Function `pc_spectral_divergence(fiber_tag)`:
- Load from `PC_PHASE10_DIR`: `"$(fiber_tag)_shaped_zsolved.jld2"` and `"$(fiber_tag)_unshaped_zsolved.jld2"`
- Extract `uomega_z` (shape [50, Nt, 1]), `sim_Dt` (picoseconds), `zsave`
- Construct frequency axis: `fs_THz = fftshift(fftfreq(Nt, 1/Float64(d_s["sim_Dt"])))`
- For each z-slice i=1:50:
  - `S_s = fftshift(abs2.(d_s["uomega_z"][i, :, 1]))` (use the actual key name from JLD2, which is `uω_z` with Unicode)
  - `S_u = fftshift(abs2.(d_u["uomega_z"][i, :, 1]))`
  - `D_z_f[i, :] = 10 .* log10.((S_s .+ 1e-30) ./ (S_u .+ 1e-30))`
- Find z_diverge_3dB: first z where `maximum(abs.(D_z_f[i, :]))` exceeds 3.0 dB
- Save to `PC_RESULTS_DIR/spectral_divergence_$(fiber_tag).jld2` with keys: `D_z_f`, `fs_THz`, `zsave`, `z_diverge_3dB`, `fiber_tag`
- Return named tuple

**CRITICAL NOTE on JLD2 key names:** The Phase 10 JLD2 files use the Unicode key `uω_z` (with omega symbol), NOT `uomega_z`. Access as `d["uω_z"]` in Julia. Same for `ut_z`.

**Section H: H2 Verdict (per D-07)**

Function `pc_h2_shift_scale_characterization()`:
- Load `perturbation_smf28_canonical.jld2` and `perturbation_hnlf_canonical.jld2` from `PC_PHASE10_DIR`
- Extract shift_THz and J_shift_dB arrays
- Fit a parabola `J(df) = J_0 + a * df^2` to the 3 central points (df = -1, 0, +1 THz) to estimate 3 dB tolerance width
- 3 dB tolerance = sqrt(3/a) THz (the df at which the parabola drops 3 dB from optimum)
- Compare with Raman gain bandwidth (13.2 THz): compute ratio = tolerance / 13.2 THz
