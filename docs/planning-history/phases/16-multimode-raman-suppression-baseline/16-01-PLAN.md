### T2 — `scripts/mmf_setup.jl` (new)

Function `setup_mmf_raman_problem(; preset=:GRIN_50, L_fiber=1.0, P_cont=0.05, pulse_fwhm=185e-15, pulse_rep_rate=80.5e6, Nt=2^13, time_window=10.0, mode_weights=nothing, raman_threshold=-5.0, λ0=1550e-9)` returning `(uω0, fiber, sim, band_mask, Δf, raman_threshold, mode_weights)`.

Internally:
- Looks up the preset.
- Calls `MultiModeNoise.get_disp_sim_params(λ0, M, Nt, time_window, β_order)` — reuses the existing sim param builder (no mods).
- Calls `get_params(f0, c0, nx, spatial_window, radius, core_NA, alpha, M, Nt, Δt, β_order; Δf)` from `src/simulation/fibers.jl` to get `βn_ω, Dω, γ, ϕ, x`.
- Builds the MMF `fiber` dict in the same shape expected by `solve_disp_mmf`:
  - `fiber["L"] = L_fiber`
  - `fiber["Dω"] = Dω`
  - `fiber["γ"] = γ`
  - `fiber["hRω"]` — built via `MultiModeNoise.get_raman_response(sim)` or equivalent existing helper; if none, construct the Blow-Wood Raman response inline (same formula used by `get_disp_fiber_params_user_defined`). *Check at implementation time which helper is actually public.*
  - `fiber["one_m_fR"] = 1 - fR`
  - `fiber["zsave"] = nothing` (the optimizer does not need per-z save).
  - `fiber["fR"] = fR`
- Normalizes `mode_weights` to unit norm if provided, else defaults to `MMF_DEFAULT_MODE_WEIGHTS / norm`.
- Calls `MultiModeNoise.get_initial_state(mode_weights, P_cont, pulse_fwhm, pulse_rep_rate, "sech_sq", sim)` to get `(ut0, uω0)`.
- Builds `band_mask = fftfreq(Nt, 1/sim["Δt"]) .< raman_threshold`.

Validates: all array shapes (Nt,M) or (Nt,M,M,M,M), total input energy finite, `issubtype(Dω, Matrix)`.

No time-window auto-sizing in this helper — documented as caller responsibility. (Auto-sizing is SMF-only because it uses a scalar β₂; at MMF we have a per-mode β₂.)

Acceptance: Call the helper, check that `solve_disp_mmf(uω0, fiber, sim)` returns a valid solution at L=0.5m without error.

### T3 — `src/mmf_cost.jl` (new)

Three cost functions, all with signature `(uωf::AbstractMatrix, band_mask::AbstractVector) -> (J, dJ)`:

```
mmf_cost_sum(uωf, band_mask)        # Σ_m E_band_m / Σ_m E_total_m  (= spectral_band_cost up to isomorphism)
mmf_cost_fundamental(uωf, band_mask) # E_band_1 / E_total_1
mmf_cost_worst_mode(uωf, band_mask)  # max_m (E_band_m / E_total_m), smooth-max w/ log-sum-exp for gradients
```

Each returns `J::Float64` and `dJ::Matrix{ComplexF64}` with shape `(Nt, M)` representing `∂J/∂conj(uωf)` — same sign/normalization convention as `spectral_band_cost`. For `mmf_cost_worst_mode`, use log-sum-exp with τ=50 for a smooth differentiable max; document the tradeoff.

Include guard `_MMF_COST_JL_LOADED`.

Acceptance: For M=1 and uniform mode content, `mmf_cost_sum == mmf_cost_fundamental == mmf_cost_worst_mode` to 1e-12.

### T4 — `scripts/mmf_raman_optimization.jl` (new)

Defines:
- `cost_and_gradient_mmf(φ::Vector{Float64}, c_m::Vector{<:Complex}, uω0_base, fiber, sim, band_mask; variant=:sum, λ_gdd=0.0, λ_boundary=0.0, log_cost=true)` — φ is SHARED across modes.
  - Expands `φ` to `(Nt, M)` via broadcast.
  - Applies `uω0_shaped = uω0_base .* c_m' .* cis.(φ)` — note the broadcast of `c_m` across the time axis.
  - Calls `solve_disp_mmf`, gets `ũω`, lab-frame at L via `cis.(Dω * L) .* ũω(L)`.
  - Computes `J, λωL = <variant>_cost(uωf, band_mask)`.
  - Calls `solve_adjoint_disp_mmf(λωL, ũω, fiber, sim)`, gets `λ₀`.
  - Gradient `∂J/∂φ_expanded = 2 .* real.(conj.(λ₀) .* (1im .* uω0_shaped))` shape (Nt, M), then **reduce** `∂J/∂φ = sum(∂J/∂φ_expanded, dims=2)[:]` — the physically-realizable gradient on the shared phase.
  - Optional log-scaling matching `raman_optimization.jl::cost_and_gradient`.
  - GDD and boundary penalties optional; apply on `φ::Vector` (not (Nt, M)).
  - Returns `(J_total, grad_total::Vector{Float64})`.
- `optimize_mmf_phase(uω0_base, c_m, fiber, sim, band_mask; φ0=nothing, max_iter=30, variant=:sum, log_cost=true, store_trace=true, seed=42)` — L-BFGS driver via Optim.jl, mirrors `optimize_spectral_phase`.
- `plot_mmf_result(φ_before, φ_after, uω0_base, c_m, fiber, sim, band_mask, Δf, raman_threshold; save_prefix)` — produces:
  - `<prefix>_total_spectrum.png`: before/after output spectrum (sum-over-modes) vs frequency, with Raman band highlighted.
  - `<prefix>_per_mode_spectrum.png`: one panel per mode, before/after.
  - `<prefix>_phase.png`: before (flat) and after (gauge-fixed polynomial-removed) φ(ω).
  - `<prefix>_convergence.png`: J_dB vs L-BFGS iteration.
- Tail guard: script only runs if `abspath(PROGRAM_FILE) == @__FILE__` (pattern from existing scripts).

Acceptance: `julia -t auto scripts/mmf_raman_optimization.jl` at the canonical config converges (J decreases monotonically; final J < 0.5 × initial J).

### T5 — `test/test_phase16_mmf.jl` (new)

Four testsets:
1. **Energy conservation** — At GRIN-50, L=0.5m, M=6, LP01-only launch, run `solve_disp_mmf`. Check `|Σ|uωf|² - Σ|uω0|²| / Σ|uω0|² < 1e-4` (attenuator loss only, not Raman energy loss which would fail this bound on the total).
   - *Note:* Raman is DISSIPATIVE in the sense that the convolution `hRω * δKt` can move energy out of the optical field into the phonon bath via the imaginary part of `hRω`. So the tolerance is `5e-3` at short L, `2e-2` at L=1m. Calibrate from a single run first, then bake in the constant.
2. **M=1 limit** — Call `cost_and_gradient_mmf` at M=1 with LP01-only launch vs `raman_optimization.jl::cost_and_gradient` at the matched SMF preset. Expect:
   - `|ΔJ_dB| < 0.5 dB`
   - `max|Δ∇J_φ| / max|∇J_φ| < 5e-3`
   - Tested at 3 random φ seeds.
   - *Caveat:* the SMF path uses `get_disp_fiber_params_user_defined` which gives slightly different Dω than `get_params` at M=1 (scalar vs matrix, different β₀/β₁ reference). For this test, construct a matched fiber via `get_disp_fiber_params_user_defined` with the SMF-28 parameters — NOT via `get_params`. Document that the M=1-limit test validates the COST/GRADIENT layer, not the fiber construction.
3. **Finite-difference gradient check** — 5 random φ components at M=6, GRIN-50, L=0.3m (small to keep runtime low). `(J(φ+ε·eᵢ) - J(φ-ε·eᵢ)) / (2ε)` vs the analytic gradient. Expect `relative error < 1e-4` at ε=1e-5.
4. **Shape sanity** — φ is `Vector{Float64}` of length Nt, `∂J/∂φ` is a `Vector{Float64}` of length Nt (not (Nt, M)).

Acceptance: 4/4 testsets pass on the burst VM.
