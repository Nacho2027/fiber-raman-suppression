## Tasks

### T1 — Implement `scripts/multivar_optimization.jl`

Create the main script. Contents:
- Top-of-file docstring: purpose, variables supported, entry points, cross-ref
  to derivations and schema docs.
- Include guard `_MULTIVAR_OPT_LOADED`.
- `include("common.jl")`, `using Optim, JLD2, JSON3, FFTW, Printf, Logging, …`.
- `@kwdef mutable struct MVConfig` — preconditioning scales, regularizer
  weights, variable tuple.
- `cost_and_gradient_multivar(x, uω0, fiber, sim, band_mask, cfg)` — assembles
  `u_shaped` from the unified `x`, does ONE forward-adjoint solve, returns
  `(J, g, diagnostics)` where `g` has the same flat layout as `x`.
- `unpack_multivar(x, cfg)` / `pack_multivar(φ, A, E, c, cfg)` — layout
  marshalling per decisions D5 layout.
- `optimize_spectral_multivariable(uω0, fiber, sim, band_mask; …)` — L-BFGS
  wrapper using `only_fg!`. Supports `variables` kwarg to select subset.
- `save_multivar_result(prefix, result, meta)` — JLD2 + JSON sidecar.
- `load_multivar_result(prefix)` — inverse.
- `run_multivar_optimization(; kwargs...)` — high-level runner analogous to
  `run_optimization` in `raman_optimization.jl`.

**Guardrails:**
- All gradient code asserts `all(isfinite, g)` postcondition.
- Box-constrained optimizer (Fminbox) if amplitude is enabled (δ_bound=0.10);
  otherwise plain LBFGS.
- `log_cost=true` default.
- Mode-coefficient variable emits `@warn` and is stripped from `variables`.

### T2 — Implement `scripts/test_multivar_gradients.jl`

Test file. Contents:
- Load `multivar_optimization.jl`.
- For each `variables` tuple in `[(:phase,), (:amplitude,), (:phase, :amplitude),
  (:phase, :amplitude, :energy)]`:
  - Set up a small problem (Nt=2^12, L=1m, P=0.1W, SMF-28).
  - Draw a random non-zero test point.
  - Run `cost_and_gradient_multivar` to get adjoint `g`.
  - For 3 random indices per block (phase/amp/energy/…), compute finite-diff
    gradient with ε tuned per block (ε_φ=1e-5, ε_A=1e-6, ε_E scaled).
  - Assert `rel_err ≤ 1e-6`.
- Round-trip save/load:
  - Save a mock `result` struct → temp prefix.
  - Load it → assert bit-identical arrays.
  - Assert both `.jld2` and `.json` files exist.
- Test prints `@info` per case; exits nonzero on any failure.

|---|---|
| Gradient validation fails at 1e-6 for amplitude | Re-derive §4 with the A=0 edge case, use sensor-based FD step |
| Amplitude optimization causes cost singularity at A→0 | `Fminbox` with `A ∈ [1-δ, 1+δ]` prevents zero-amplitude trivial sol. |
| L-BFGS stalls on mixed-scale gradient | Preconditioning (D5) implemented from day one |
| Demo run slower than expected on burst VM | Single problem, ~2 minutes expected; burst lock held for ≤ 10 min |
| Session collision on `.planning/ROADMAP.md` | Append-only (P3); do not edit existing rows |
