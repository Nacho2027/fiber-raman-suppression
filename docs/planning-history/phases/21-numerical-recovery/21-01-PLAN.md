## Task 1: Ship the recovery helpers

Acceptance:
- Script is self-contained, uses existing shared files read-only, and contains
  no shared-file edits.

## Task 2: Recover Sweep-1

Create `scripts/recovery_sweep1.jl`.

Inputs:
- `results/raman/phase_sweep_simple/sweep1_Nphi.jld2`

Method:
- Load the 7 target `N_phi` points: `4, 8, 16, 32, 64, 128, 16384`.
- Determine one honest grid for the shared config `SMF-28, L=2m, P=0.2W`
  using the flat pulse and all 7 stored `phi_opt` seeds.
- Re-run each case independently in parallel with `Threads.@threads`.
- Use low-resolution optimization for `N_phi < Nt` and the full optimizer for
  `N_phi = Nt`.
- Emit a standard image set per recovered phase.

Acceptance:
- Every case reports `J_dB`, iteration count, edge fraction, and image paths.
- Aggregate markdown explicitly answers whether the original knee story survives.

## Task 3: Normalize Session F 100 m schema

Create `scripts/recovery_validate_100m_schema.jl`.

Method:
- Inspect `results/raman/phase16/100m_validate_fixed.jld2`.
- Discover the stored keys and extract the phase / metrics needed for honest
  validation.
- Reconstruct the intended simulation config (`SMF-28`, `L=100m`, `P=0.05W`,
  `Nt=32768`, `T=160ps`, `beta_order=2`) and forward-validate the stored phase.
- Emit the standard image set for the stored 100 m phase.
- Write a normalized Phase-21 markdown report stating whether the result is
  RECOVERED or only a lower bound.

Acceptance:
- The recovered report states both the honest `J_dB` and whether the stored run
  had `converged=true/false`.

## Task 4: Re-anchor Phase 13

Create `scripts/reanchor_hessian.jl`.

Inputs:
- `results/raman/phase13/hessian_smf28_canonical.jld2`
- `results/raman/phase13/hessian_hnlf_canonical.jld2`

Method:
- For each config, determine an honest grid using the original `phi_opt`.
- Warm-start the original phase onto that grid.
- Re-run L-BFGS in parallel across the two configs.
- Emit standard images and per-config markdown.

Acceptance:
- Each config produces a new honest `J_dB`, final edge fraction, and a verdict:
  stationary point persisted / moved materially / failed to recover.
