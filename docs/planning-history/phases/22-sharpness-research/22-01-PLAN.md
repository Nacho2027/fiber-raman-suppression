# Plan 22-01 — Implement sharpness-aware sweep and synthesize the depth/robustness Pareto

<objective>
Build session-owned tooling that compares three sharpness-aware objectives
against the current baseline on two operating points, generates the mandatory
standard image set for every `phi_opt`, computes `sigma_3dB` and Hessian
eigenspectra, and writes a Summary that answers the real engineering question:
"should sharpness be the new default cost for a real phase shaper?"
</objective>

<must_haves>
- No edits outside the Phase 22 directory and `scripts/sharpness_*`.
- Use existing Phase 13/14 code paths read-only; do not rebuild shared
  HVP, eigenspectrum, or standard-image logic.
- Deterministic stochastic objectives: fixed probe/noise banks per run.
- Thread over independent runs with `deepcopy(fiber)` per task.
- Save a standard image set for every baseline and every optimized `phi_opt`.
- Produce:
  - one machine-readable JLD2 result bundle
  - one Pareto plot PNG
  - one Summary markdown with a verdict paragraph
</must_haves>

<tasks>

<task id="22-01-T1" type="execute" autonomous="true">
<title>Implement the shared Phase 22 library</title>

<action>
Create `scripts/lib.jl` with:

- operating-point builders for OP-A and OP-B
- warm-start loaders for the Phase 17 baseline and Session E Pareto point
- control-space abstraction `phi = A*x`
- objective wrappers for:
  - baseline `J_dB`
  - SAM
  - Hessian-trace penalty
  - Monte Carlo Gaussian average
- post-hoc metric helpers for:
  - `J_dB_plain`
  - `sigma_3dB`
  - Hessian eigenspectrum / indefiniteness
  - `save_standard_set`

Read-only includes are allowed from:

- `common.jl`
- `raman_optimization.jl`
- `determinism.jl`
- `standard_images.jl`
- `sharpness_optimization.jl`
- `hvp.jl`
- `hessian_eigspec.jl`
- `sweep_simple_param.jl`
</action>

<acceptance_criteria>
- A small smoke invocation can build both operating points and load both
  warm starts without error.
- The low-resolution operating point reconstructs a length-`Nt` `phi_opt`
  from the saved `c_opt`.
- No shared file is modified.
</acceptance_criteria>
</task>

<task id="22-01-T2" type="execute" autonomous="true" depends_on="22-01-T1">
<title>Implement the threaded production runner</title>

<action>
Create `scripts/run.jl` that:

- enumerates the 26 records (2 baselines + 24 flavor runs)
- supports `--smoke`
- executes independent production runs under `Threads.@threads`
- snapshots per-run results and a consolidated `phase22_results.jld2`
- writes standard images under
  `.planning/phases/22-sharpness-research/images/`

Each task must use `deepcopy(fiber)` and keep FFTW single-threaded.
</action>

<acceptance_criteria>
- `--smoke` completes locally.
- Production mode writes one consolidated results file and per-run image
  folders/tags.
- Failures are captured in the result bundle instead of aborting the whole
  batch.
</acceptance_criteria>
</task>
