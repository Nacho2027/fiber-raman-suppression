# Phase 27 — Plan 01

**Title:** Numerical analysis audit + CS 4220 application report  
**Status:** Complete  
**Owner:** `sessions/numerics`

## Goal

Produce a numerics-focused report that crosswalks the Cornell CS 4220 Spring
2026 course material against this Raman-suppression codebase, identifies the
most important current blockers and opportunities, and plants seeds for any
future work large enough to deserve its own phase.

## Scope

- In scope:
  - `.planning/phases/25-*`
  - `.planning/seeds/*.md` created by this audit
  - roadmap/state updates limited to Phase 27 bookkeeping
- Out of scope:
  - `src/**` refactors
  - shared-script edits
  - heavy compute reruns
  - paper/docs rewrites outside `.planning/**`

## Tasks

1. Read the relevant CS 4220 course material and extract the topics that are
   materially relevant to this project.
2. Audit the current codebase and planning state for numerical blockers,
   missing trust infrastructure, and already-existing numerical assets.
3. Write a report that distinguishes:
   - immediate recommendations,
   - medium-sized improvements,
   - and future-phase candidates.
4. Plant seeds for the future-phase candidates.
5. Leave the code untouched outside `.planning/**`.

## Verification

- The report names specific CS 4220 topics and maps them to concrete repo
  opportunities or blockers.
- The report calls out planning/codebase drift with evidence.
- The audit does not propose already-fixed issues as open work.
- Seeds exist for the largest next-step ideas.

## Risks

- Overfitting the audit to whatever is currently most visible in planning docs.
  Mitigation: include direct code inspection and the external course material.
- Turning the audit into abstract "math brainstorming."
  Mitigation: every recommendation must be tied back to a real file, workflow,
  or failure mode in this repo.
