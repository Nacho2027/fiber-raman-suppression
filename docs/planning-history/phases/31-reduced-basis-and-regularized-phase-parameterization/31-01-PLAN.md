<objective>
Build the Phase 31 library (basis + penalty) with unit tests, then execute Branch A — the reduced-basis sweep — on the burst VM. Branch A tests whether explicit basis restriction (polynomial, chirp_ladder, DCT, cubic, linear at N_phi ∈ {3..256}) recovers competitive Raman suppression at substantially lower dimensionality than the full Nt=2^14 grid, while enabling saddle-masking diagnostics (Phase 35 pitfall) and interpretability metrics (Phase 13 polynomial projection).

Purpose: Extend existing basis infrastructure per locked decision D-01 + D-02. Keep basis restriction and penalty regularization separate per D-03. Record the simplicity/robustness axes that matter per D-04. The phase's core scientific claim hinges on whether `:polynomial` at order ≥ 6 or `:dct` at `N_phi = 32-64` reaches within ~10 dB of the full-grid optimum while giving a PSD restricted Hessian that is NOT just basis-masking.

Output: Two library files (phase31_basis_lib.jl, phase31_penalty_lib.jl), one driver (phase31_run.jl), one test file (test_phase31_basis.jl), and 21 JLD2 rows + 21 standard-image sets under `results/raman/phase31/sweep_A*`. Plan 02 consumes these libraries and JLD2 rows to run Branch B + transfer + analysis.
