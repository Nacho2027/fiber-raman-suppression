# Phase 35 — Plan 01

**Title:** Reduced-basis Hessian ladder + negative-curvature escape study  
**Status:** In progress  
**Owner:** `sessions/saddle-escape`

## Goal

Determine whether genuine local minima exist in reachable Raman-suppression
territory and, if they do, whether negative-curvature-aware escape from the
current L-BFGS saddles can reach them.

## Scope

- In scope:
  - `scripts/saddle_phase35_*`
  - `results/raman/phase35/**`
  - `.planning/phases/35-saddle-escape`
- Out of scope:
  - edits to shared source or shared scripts
  - new optimizer integration into the main production path
  - large full-resolution Hessian reruns beyond what Phase 13 already produced

## Tasks

1. Build a Phase 35 runner that loads the canonical `N_phi` sweep data,
   reconstructs the low-resolution baselines, and computes control-space
   Hessian spectra for the usable `N_phi` ladder.
2. Classify each baseline as minimum-like or saddle-like and summarize how that
   classification changes with control-space dimension and achieved `J_dB`.
3. Select one or more competitive indefinite saddles and run a signed
   negative-curvature escape study:
   - line scan along the most negative eigenvector,
   - reoptimize from escaped starts,
   - save standard-image sets for every fresh `phi_opt`.
4. Write the final Phase 35 report:
   - whether genuine minima exist in reachable `J` territory,
   - what the best evidence says about reachability,
   - and which optimizer path should be recommended next.
5. Record review and verification artifacts under the phase directory.

## Verification

- The `N_phi` ladder table includes `J_dB`, `lambda_min`, `lambda_max`, and an
  `indefinite` classification for each analyzed baseline.
- At least one negative-curvature escape produces fresh optimization runs with
  mandatory standard images on disk.
- The final report makes a binary recommendation on the advisor question:
  whether to frame the problem as "minima exist but are hard to reach" or
  "competitive solutions are intrinsically saddle-dominated."

## Risks

- Dense Hessian computation could be too slow at the largest `N_phi` values.
  Mitigation: keep the ladder adaptive and prioritize the dimensions closest to
  the good-Raman regime.
- Some historical artifacts may be missing from this checkout.
  Mitigation: rebuild the needed low-resolution baselines from the local
  sweep bundle rather than depending on prior per-run outputs.
