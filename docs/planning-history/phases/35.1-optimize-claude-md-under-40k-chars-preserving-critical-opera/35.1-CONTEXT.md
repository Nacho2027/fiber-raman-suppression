<decisions>
## Implementation Decisions

### MUST PRESERVE (verbatim or with only surface-level wording compression)
- Project header and Core Value statement
- Standard output images mandate (the `save_standard_set(...)` rule and the template)
- Constraints block (tech stack, backward compatibility, output format, performance)
- **GSD Workflow Enforcement** section — strict-mode rule, allow-list, routing commands
- **Codex Runtime Constraints** section — whitelist, blacklist, escape-hatch phrase, verification script reference
- **Multi-Machine Workflow** section — session hygiene (start/end), `.planning/` sync rules, machine inventory table, common pitfalls
- **Parallel Session Operation Protocol** — Rules P1 through P7 in full force (file namespaces, branch-per-session, append-only shared files, sync semantics, burst-VM wrapper, host distribution, integration checkpoints)
- **Running Simulations — Compute Discipline** — Rules 1, 2, 3 in full (burst-VM mandate, threading mandate, stop-VM mandate), plus the `deepcopy(fiber)` pattern and the pre-simulation checklist
- SI Units Convention block (physics correctness)
- Unit conventions that any future agent needs to reconstruct simulation state

### MAY TRIM (these are safe to compress or relocate)
- **Technology Stack** verbose subsections — keep a compact table, drop redundant bullet expansions (versions, stdlib callouts already present in Project.toml)
- **Conventions / Naming** — collapse long example lists to a short principle + one example
- **Architecture / Layers / Key Abstractions** — reduce to essentials; long layer descriptions are derivable from source
- **Design Decisions** narrative — keep bullets, drop prose around each
- Repeated example code blocks when the same mechanism is documented in a referenced file (e.g., `scripts/burst/README.md` already documents the full burst VM mechanism — CLAUDE.md can point to it instead of duplicating)
- Section header decorations (`---`, horizontal rules) where they add length without value
- Redundant "see X" callouts that appear twice
- Historical "as of DATE" prose where the date has passed and the rule is now the default

### MUST NOT INTRODUCE
- No new sections
- No new rules (scope is compression, not policy change)
- No removal of any rule even if it feels redundant — if two sections overlap, merge wording but keep every directive
- No change to any exact phrase that Codex is instructed to return verbatim ("This task requires the `[name]` skill…")

### Claude's Discretion
- Exact wording choices when compressing prose
- Which redundant examples to remove vs compress
- Whether to introduce a compact table to replace a bulleted prose list
- Ordering of subsections within a preserved section (ordering may change for density if meaning is preserved)

</decisions>
